# GitHub Basics Exercise

## Objective

Learn the fundamentals of Git and GitHub, including repository creation, branching, committing changes, merging branches, and pushing code to GitHub.

---

## Exercise Steps

### 1. Create a Local Repository

Create a new repository locally on your machine.

---

### 2. Create a GitHub Repository

Create a new GitHub repository. Name it whatever you want.

---

### 3. Connect Local Repository to GitHub

Connect your local repository to the GitHub repository you created.

---

### 4. (Optional) Rename the Default Branch

Rename the default branch from `master` to `main`.

---

### 5. Create `preferences.txt`

* Create a new file called `preferences.txt`.
* Leave it empty.
* Make your first commit on the `main` branch.

---

### 6. Push the Main Branch to GitHub

* Push your `main` branch to GitHub.
* Verify that the empty `preferences.txt` file is visible on GitHub.

---

### 7. Create New Branches

Create two new branches:

* `foods`
* `movies`

---

### 8. Update the Foods Branch

* Switch to the `foods` branch.
* Add three (or more) of your favorite foods to `preferences.txt`.
* Stage and commit your changes on the `foods` branch.

---

### 9. Update the Movies Branch

* Switch to the `movies` branch.
* Add three (or more) of your favorite movies to `preferences.txt`.
* Stage and commit your changes on the `movies` branch.

---

### 10. Push the Foods Branch

* Push the `foods` branch to GitHub.
* Confirm that the branch appears on GitHub.

---

### 11. Push the Movies Branch

* Push the `movies` branch to GitHub.
* Confirm that the branch appears on GitHub.

---

### 12. Merge Branches into Main

* Merge the `foods` branch into the `main` branch.
* Merge the `movies` branch into the `main` branch.
* If merge conflicts occur, resolve them so that both favorite foods and favorite movies appear together in `preferences.txt`.

---

### 13. Push Final Changes

* Push the latest changes from the `main` branch to GitHub.

---

## Expected Outcome

By completing this exercise, you should be able to:

* Create and initialize Git repositories.
* Connect local repositories to GitHub repositories.
* Create and work with branches.
* Commit changes to different branches.
* Push branches to GitHub.
* Merge branches into the main branch.
* Resolve merge conflicts.
* Verify changes on GitHub.
